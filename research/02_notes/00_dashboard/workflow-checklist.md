# Workflow Checklist

这份清单只保留日常会反复使用的动作。

## 每次开始前

- [ ] 打开 `start-here.md`
- [ ] 明确今天只推进一个主问题或一个方法点

## 文献流转

- [ ] 新文献先进入 `Zotero`
- [ ] 高价值文献再写入 `01_literature_notes/`
- [ ] 至少连接 1 张方法卡片
- [ ] 至少连接 1 条问题笔记

## 方法学习

- [ ] 每次只推进一个方法对象
- [ ] 优先配一个最小例子或脚本
- [ ] 把“仍然不清楚”的点写回方法卡片

## 研究沉淀

- [ ] 每次工作至少产出一种资产：
  - 文献笔记
  - 方法卡片
  - 问题笔记
  - 数据卡片
  - 研究备忘录

## 每周收口

- [ ] 更新一份周回顾
- [ ] 整理 `Zotero`
- [ ] 整理 `Obsidian`
- [ ] `git add -> git commit`

## 暂时不要优先做的事

- [ ] 不优先折腾插件
- [ ] 不优先下载大体量数据库
- [ ] 不优先做复杂自动化
- [ ] 不优先系统补数学基础
