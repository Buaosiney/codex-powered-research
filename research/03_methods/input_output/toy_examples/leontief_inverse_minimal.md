# Leontief 逆矩阵最小数值例子

## 目标

用一个两部门的最小例子理解：

- 什么是直接消耗系数矩阵 `A`
- 为什么要计算 `(I - A)^(-1)`
- 它如何把最终需求转化为总产出需求

## 场景设定

假设经济中只有两个部门：

1. 能源部门
2. 制造部门

直接消耗系数矩阵设为：

```text
A = | 0.20  0.10 |
    | 0.30  0.20 |
```

解释：

- 制造部门每生产 1 单位产出，需要 0.10 单位能源投入和 0.20 单位制造投入
- 能源部门每生产 1 单位产出，需要 0.20 单位能源投入和 0.30 单位制造投入

设最终需求为：

```text
y = | 100 |
    |  50 |
```

表示：

- 对能源的最终需求为 100
- 对制造的最终需求为 50

## 第一步：构造 `I - A`

单位矩阵：

```text
I = | 1  0 |
    | 0  1 |
```

所以：

```text
I - A = | 0.80  -0.10 |
        | -0.30  0.80 |
```

## 第二步：求 Leontief 逆矩阵

```text
L = (I - A)^(-1)
```

对这个 2x2 矩阵计算后可得：

```text
L ≈ | 1.3115  0.1639 |
    | 0.4918  1.3115 |
```

## 第三步：计算总产出

```text
x = Ly
```

代入后：

```text
x ≈ | 139.34 |
    | 114.75 |
```

## 如何解释结果

这意味着：

- 为了满足 `100` 单位能源最终需求和 `50` 单位制造最终需求
- 经济系统中实际需要生产大约：
  - 能源：`139.34`
  - 制造：`114.75`

多出来的部分不是“浪费”，而是产业链中间投入引致的额外生产需求。

## 关键直觉

1. 最终需求不是总产出
   因为生产最终产品还需要中间投入

2. Leontief 逆矩阵把直接和间接需求都算进去了
   这就是它在 IO 研究中的核心价值

3. 以后做环境扩展时，只要再乘上环境强度，就能把总产出需求转成总排放或总资源使用

## 与你的主线的关系

- 这是理解环境扩展 IO 的最低门槛
- 后面分析投资、资本形成和基础设施时，所有“上游带动效应”都建立在这种结构直觉上
